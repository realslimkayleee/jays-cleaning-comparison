# Jay's Cleaning Services - Presentation Comparison

This directory contains the presentation landing page and floating back bar designed to showcase three concept directions to Jay's Cleaning Services.

## Structure
- `index.html`: The main comparison page.
- `assets/nav-bar.js`: The floating return bar script.
- `assets/nav-bar.css`: The styling for the floating return bar.
- `assets/favicon.svg`: The presentation site favicon.
- `concept-a/`, `concept-b/`, `concept-c/`: The three standalone HTML concepts generated by Stitch.

## Integration & Setup
1. **Drop the concept files:** The concept HTML files should reside at `concept-a/index.html`, `concept-b/index.html`, and `concept-c/index.html`.
2. **Inject the script:** Ensure the following script is present inside each concept's `index.html` file (right before `</body>`). 
   (Note: I have already injected these for you in this codebase, but if you update the concepts later, you will need to re-add them.)

   **For Concept A:**
   ```html
   <script src="../assets/nav-bar.js"
           data-concept-name="Reset"
           data-concept-letter="A"
           data-back-href="../index.html"
           data-choose-href="mailto:designer@example.com?subject=Jay%27s%20—%20I%20choose%20Concept%20A"
           defer></script>
   ```

   **For Concept B:**
   ```html
   <script src="../assets/nav-bar.js"
           data-concept-name="Crew"
           data-concept-letter="B"
           data-back-href="../index.html"
           data-choose-href="mailto:designer@example.com?subject=Jay%27s%20—%20I%20choose%20Concept%20B"
           defer></script>
   ```

   **For Concept C:**
   ```html
   <script src="../assets/nav-bar.js"
           data-concept-name="Regeneration"
           data-concept-letter="C"
           data-back-href="../index.html"
           data-choose-href="mailto:designer@example.com?subject=Jay%27s%20—%20I%20choose%20Concept%20C"
           defer></script>
   ```

3. **Update Designer Info:** Open `index.html` and search for `designer@example.com` and `[Designer Name]`. Update these to your actual email address and name so the "I want this one" buttons route to your inbox correctly.

## Deployment
Because this site contains no frameworks, build steps, or backend dependencies, you can deploy it directly as a static site to any host:
- **Netlify/Vercel/Cloudflare Pages:** Drag and drop this folder directly into their dashboard to deploy instantly.
- **GitHub Pages:** Commit this directory and enable GitHub Pages on the `main` branch.
- **FTP:** Upload the entire folder directly to a standard web host.
